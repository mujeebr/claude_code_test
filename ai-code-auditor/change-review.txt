  POTENTIAL BUGS
============================================================

• Line 29: `echo "\n\nProvide suggestions..."` — in bash, \n inside double quotes
  is NOT a newline; it prints literally as \n. Use $'...' syntax or printf instead:
    printf "\n\nProvide suggestions, bugs, and improvements.\n" >> prompt.txt

• Line 23: `git diff origin/main > diff.txt` — this diffs the working tree against
  origin/main, not the PR branch commits. For a PR review, use:
    git diff origin/main...HEAD > diff.txt
  The three-dot syntax correctly captures only the PR's changes.

• No error handling — if `git fetch` or `git diff` fails, subsequent steps will
  silently operate on an empty or stale diff.txt.
